# Angular Debugging & Browser Sync

- **Runtime First**: Before fixing runtime errors, always use Browser Tool to check http://localhost:4200.
- **Console Monitoring**: When problems occur, analyze the console output for errors Zone.js, Dependency Injection, or Hydration.
- **Visual Validation**: After changing CSS or HTML templates, take a screenshot via the browser to confirm the correctness of the rendering.
- **Angular Context**: When analyzing bugs, consider the current state of signals (Signals) and the component lifecycle.
